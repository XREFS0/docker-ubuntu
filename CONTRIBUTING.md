# Contributing Guidelines

First off, thank you for considering contributing to this project! It's people like you that make it such a great tool.

## How to Contribute

### 1. Reporting Bugs
If you find a bug, please create an issue providing as much detail as possible. Include:
* Your operating system and browser version.
* Steps to reproduce the bug.
* Expected behavior vs. actual behavior.

### 2. Suggesting Enhancements
Enhancement suggestions are highly welcomed! When creating an issue, please explain:
* The current behavior.
* The new, expected behavior.
* Why this change would be useful to most users.

### 3. Pull Requests
* Fork the repository and create your branch from `main`.
* If you've added code that should be tested, add tests.
* Ensure your code passes all existing tests and linters.
* Issue that pull request!

## Code of Conduct
By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).
